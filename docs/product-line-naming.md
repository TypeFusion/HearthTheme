# Product Line Naming

Updated: 2026-04-06

## Goal

Keep `HearthCode` fixed as the brand.

Treat public line names as changeable.

## Rule

Line names should describe a color world.

They should not describe:

- a workflow
- a user skill level
- a task type
- a promise like "focus" or "productivity"

## Current Working Names

- `Ember`
- `Moss`

These are current public working names, not permanent final names.

## What A Good Line Name Should Do

- feel short and clean in a theme picker
- work in both dark and light
- describe taste, not function
- fit under the `HearthCode` brand and still read cleanly in a theme picker

## What Should Stay Stable

- brand name: `HearthCode`
- internal scheme ids: `hearth`, `signal`

Internal ids can stay stable even if public line names change later.

## Rename Timing

Do not rename line names while the palettes are still moving.

Rename only after:

- both lines feel visually mature
- both lines are clearly equal in capability
- a new name feels more precise than the working name it replaces
