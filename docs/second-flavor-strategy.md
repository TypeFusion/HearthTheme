# Second Flavor Strategy

Updated: 2026-04-05

## 1) Working Direction

Working codename: `Moss`

This is not the final product name.
It is a temporary direction label for a second flavor that should feel clearly different from `HearthCode`.

The direction is no longer "higher-energy Signal."

The new target is:

- retro, but refined
- charcoal-led, not chocolate-washed
- structurally clearer than Hearth
- more articulated in syntax lanes than in background tint

If `HearthCode` is tuned for steady reading, long sessions, and warm composure, the second flavor should feel more editorial, more tactile, and more visibly separated in one screen.

## 2) Why Cairn Is Not Enough

`CairnCode` validates the multi-flavor pipeline, but it does not create enough product distance.

It changes material and metaphor:

- hearth -> stone
- ember -> clay
- mineral -> slate

But it does not change the energy model enough.

Both lines still feel:

- low-glare
- quiet
- long-session friendly
- grounded
- structurally restrained

That makes `Cairn` useful as an engineering proof, but weak as a second public product.

The second public-facing flavor should not merely change the texture of calm.
It should change the pace, emphasis, and emotional posture of the theme.

## 3) Product Thesis

`Moss` should be a paper-and-pigment code color language for people who want stronger yellow-green rhythm and material atmosphere without falling into novelty-theme noise.

It should feel:

- vintage but not cosplay
- colorful but not loud
- tactile but not muddy
- clearly separated but still daily-drivable

This flavor should reward users who:

- want a stronger visual identity than Hearth
- like gruvbox-adjacent material warmth
- still want obvious syntax landmarks
- prefer charcoal-and-paper atmospheres over modern graphite minimalism

## 4) Contrast With Hearth

| Axis | HearthCode | Second Flavor Direction |
| --- | --- | --- |
| emotional posture | calm, warm, stable | tactile, editorial, vintage |
| reading rhythm | continuous, low-fatigue | punctuated, more structured |
| surface priority | glare reduction first | material atmosphere plus structure |
| semantic emphasis | restrained separation | clearer split-lane syntax |
| chroma strategy | bounded and dry | selective earthy peaks |
| first impression | composed | retro and more distinctive |
| ideal use | all-day reading and coding | users wanting stronger syntax identity with warm materials |

This second flavor should be the one users choose when they want the editor to feel more designed and more visibly separated, not simply louder.

## 5) Lessons From GruvDark Theme

The `gruvdark-theme` family is useful because each variant emphasizes a different strength.

What we should borrow:

- `gruvdark`: the best base balance between charcoal substrate, parchment ink, and split syntax lanes
- `gruvdark-gbm`: the best earthy pigment richness
- `gruvdark-mono`: the best restraint and chroma discipline
- `gruvdark-tokyo`: the best cool relief and night clarity
- light variants: parchment logic instead of office-white brightness

What we should not borrow:

- exact token mapping
- all-over brown wash
- UI chrome painted with syntax hero colors
- overpacked pigment density
- novelty through saturation alone

## 6) Surface Strategy

Do not make the second flavor by tinting the whole background brown.

The substrate should be:

- charcoal first
- warm second
- clearly distinct from Hearth's softer paper warmth
- still compatible with vintage pigment families

Instead:

- use charcoal with a brown bias
- keep shells and panels slightly drier than the current `Signal` draft
- let syntax carry most of the separation work
- allow material identity, but do not let the whole page turn sepia

Recommended surface direction:

- dark climates: carbon paper, smoked wood, restrained warm charcoal
- light climates: parchment and bone paper, not modern cream office mode

The surface should establish the atmosphere, but syntax should still own the landmarks.

## 7) Semantic Role Direction

The second flavor should feel richer because the page is split into clearer syntax lanes.

Recommended lane split:

- `keyword / control`: muted blue or dusty indigo
- `function / callable`: coral or terracotta
- `type / support`: oxidized teal
- `string`: moss or dry olive-green
- `number / method`: old gold / ochre
- `property`: quieter aqua-teal support
- `operator / variable / parameter`: parchment neutrals
- `comment`: warm gray or olive-gray restraint

This is more useful than the current draft where too many roles still live in one earthy family.

The goal is not more roles with more chroma.
The goal is fewer, clearer lanes.

## 8) UI Strategy

UI chrome should stay quieter than syntax.

- buttons, badges, and app chrome should not automatically reuse the hero syntax accent
- a neutral ink or subdued blue-gray accent is often better
- syntax should be allowed to feel more designed than the shell around it

## 9) Climate Philosophy

The second flavor keeps the same four climates, but each one should preserve the same retro syntax logic.

- `dark`: the flagship. Charcoal hero, clearest split-lane syntax, strongest product identity.
- `darkSoft`: lower pressure, same pigments, no collapse into gray or brown mush.
- `light`: parchment daylight with decisive ink and visible syntax spacing.
- `lightSoft`: worn paper and softer structure, but still clearly organized.

Important constraint:

- the climates should look like one family with pressure changes, not four separate theme ideas

## 10) Anti-Goals

The second flavor should not become:

- a generic gruvbox clone
- a sepia-washed editor
- a neon contrast pack
- a colder Hearth
- a louder Cairn
- a theme where UI and syntax all fight with the same accent

It must stay mature enough for everyday work.

The right target is:

- editorial discipline
- with clearer retro syntax lanes

## 11) Reference Lenses

Useful reference lenses from mature products:

- [Catppuccin](https://github.com/catppuccin/catppuccin): separate flavors succeed when each one is a whole personality, not a small hue shift.
- [GruvDark Theme](https://github.com/darianmorat/gruvdark-theme): the most relevant reference set for substrate balance, split syntax lanes, and retro refinement.
- [gruvbox](https://github.com/morhetz/gruvbox): the original lesson in warm material worlds with strong role spacing.
- [Tokyo Night](https://github.com/tokyo-night/tokyo-night-vscode-theme): a reminder that one cool relief lane can sharpen a dark palette without making it cold.

These should be used as strategic references, not style targets.

## 12) Design Brief

Before any palette work starts, the second flavor should satisfy this brief:

1. A user should recognize in one screen that it is not `HearthCode`.
2. A user should still feel that it belongs to the same overall system discipline.
3. The difference should come mainly from substrate balance and split syntax lanes, not from painting the whole background.
4. The flavor should stay usable for real work, not just previews.
5. The default dark climate should be the hero and clearest expression of the product.

## 13) Recommended Next Step

Use the GruvDark synthesis as the tuning brief for `Signal`.

Next palette passes should prioritize:

1. less yellow in the substrate and foreground
2. a stronger split between blue/control, coral/callable, teal/type, and moss/string
3. quieter UI accents
4. keeping the light variants old-paper rather than office-white
